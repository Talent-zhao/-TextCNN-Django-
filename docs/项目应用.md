# 贴吧抑郁分析系统 — 项目应用说明

本文说明 **用户端（前台）** 的典型使用步骤，以及 **管理员端（统一后台 `admin_panel`）** 各功能入口与操作要点。默认访问地址以 `http://127.0.0.1:8000/` 为例（以实际 `runserver` 输出为准）。

**算法、融合配置与训练评估的逐步说明**：见 **`docs/算法逻辑与配置说明.md`**（新增设定须同步更新该文档 §0）。

---

## 一、用户端使用步骤

### 1.1 账号相关

| 步骤 | 说明 |
|------|------|
| 注册 | 浏览器打开 `/register/`，按页面填写信息完成注册。 |
| 登录 | 打开 `/login/`，使用 **手机号 + 密码** 登录（业务用户表以手机号为账号凭证）。 |
| 个人中心 | 登录后，导航栏右侧用户名菜单中进入 **个人中心**（`/myProfile/`），可维护资料等。 |
| 修改密码 | 菜单中 **修改密码**（`/resetPWd/`）。 |
| 注销 | 菜单中 **注销**（`/logout/`），清除前台登录 Cookie。 |

说明：**评论列表、爬取、在线预测、各类可视化图表** 等页面在代码中会读取前台 `user` 登录信息；未登录时访问可能报错或跳转，请先登录。

### 1.2 浏览评论与爬取数据

1. 登录后进入首页 **评论列表**（`/`）。
2. 打开 **爬取评论**（`/scrawl/`），按页面提示输入贴吧帖子 **tid**，提交后开始爬取。
3. 爬虫结果会写入项目根目录下的 `data/{tid}.csv`，并由后台逻辑 **导入到业务库 `Comment` 表**，供列表与图表使用。

注意：爬虫依赖贴吧页面结构与 Cookie，若站点改版可能导致失败，需调整 `app/scrawldata.py`。

### 1.3 在线预测（可选模型）

1. 打开 **在线预测**（`/predict/`）。
2. 在 **使用模型** 下拉里任选一条后台已登记且 **状态为可用（ready）** 的模型，或保留 **默认（当前启用模型）**。
3. 输入待测文本并提交；系统按该模型的类型与 `PredictService` 融合配置返回标签、综合分与风险等级等。

说明：前台可 **切换不同模型做演示**；后台 **待预测批量** 仍只使用「当前启用」的那一条模型。

### 1.4 数据可视化

在导航栏下拉菜单中进入（均需登录后使用）：

| 菜单 | 路径 | 内容概要 |
|------|------|----------|
| 初步可视化 → 词云图 | `/plot1` | 基于 `Comment` 内容词云 |
| 初步可视化 → 折线图 | `/plot2` | 微博/贴吧每日评论趋势 |
| 抑郁可视化 → 情感环形图 | `/plot3` | 情感分类分布 |
| 抑郁可视化 → 地域地图 | `/plot4` | 地域分布 |
| 抑郁可视化 → 用户评论数柱状图 | `/plot5` | |
| 抑郁可视化 → 用户等级柱状图 | `/plot6` | |
| 抑郁可视化 → 散点图 | `/plot7` | 评论长度与回复数关系 |

图表数据来源于库中 **`Comment`** 表；若为空，请先完成爬取与入库。

### 1.5 其他（可选）

- **角色调试**：`/debug/role/`（菜单中「角色调试」），用于查看当前登录用户在业务库中的 `role` 等调试信息。
- **批量情感分类**：路由 `/trigger_fenlei/`（`@check_login`），用于触发对评论的批量分类任务；一般为开发/运维使用，需在已登录前提下按项目约定方式调用（如带 CSRF 的 POST）。

## 二、管理员端说明（统一后台）

管理员端即 **`/admin_panel/` 下的业务后台**，与 Django 自带的 `/admin/`（模型注册后台）不是同一套界面。

### 2.1 登录与权限

| 方式 | 说明 |
|------|------|
| **推荐** | 使用业务库中 **`User.role = 'admin'`** 的账号，在 **前台 `/login/`** 登录后，通过导航栏 **「后台入口」** 进入后台（当前默认落地页为用户管理）。 |
| 备用 | 访问 **`/admin_panel/login/`**，使用 **Django 超级用户**（`createsuperuser`）或符合 `adminViews` 规则的 **`username=admin` 业务账号** 登录。 |

重要说明：后台各业务子页在渲染时会读取前台 **`user` Cookie** 以显示操作人、写审计与导出日志。**若仅使用 `/admin_panel/login/` 而未在前台登录，部分页面可能因缺少 `user` 信息无法正常打开。** 实际部署与演示时，建议统一采用「前台管理员账号登录 → 后台入口」。

退出专用后台登录：`/admin_panel/logout/`（清除 `admin` Cookie）。

以下路径前缀均为 **`/admin_panel/`**（另有一套无此前缀的 **兼容跳转** URL，会重定向到同名功能，书签可继续用）。

### 2.2 后台入口与导航

| 路径 | 功能 |
|------|------|
| `users/list/` | 后台默认落地页，进入用户管理。 |
| `workbench/` | 兼容地址，已下线工作台页面；访问会重定向到后台默认页。 |

旧地址 `dashboard/`、`comments/`、`statistics/` 已重定向到后台默认页，不再提供独立工作台。

### 2.3 数据管理

| 路径 | 功能 |
|------|------|
| `data/import/` | **数据导入**：上传 **CSV** 或 **JSON**，将文本写入 **`RawText`**。CSV 为表头行 + 数据行；常用字段包括 `content`（必填），以及 `id`/`external_id`、`name`/`author`、`time`/`publish_time` 等。JSON 可为数组，或 `{"data": [...]}` 形式。重复内容（按内容哈希）会跳过。 |
| `data/rawtexts/` | **原始文本列表**：关键词搜索、按状态筛选、分页；可 **变更状态**（需符合状态机规则）；支持 **按内容去重** 批量删除重复条。 |
| `data/rawtexts/<id>/` | **单条详情**：查看该条原文、清洗信息、关联标注与预测记录等。 |

状态在列表中操作时，若流转到「已清洗」等节点，系统会维护对应的 **`CleanText`** 记录（简易清洗规则以代码为准）。

### 2.4 样本标注

| 路径 | 功能 |
|------|------|
| `annotation/list/` | **标注列表**：按状态（默认待标注）、关键词筛选；对单条或批量选择 **标签** 提交标注；可初始化 **二分类 / 四分类** 标签集（接口动作 `init_binary` / `init_four`）。标注成功后，符合条件的 **`RawText`** 会进入 **待预测** 状态。 |

标签定义存于 **`LabelDefinition`**，与风险等级等字段联动，具体预设见 `app/annotationViews.py` 中 `LABEL_PRESETS`。

### 2.5 预测与预警

| 路径 | 功能 |
|------|------|
| `risk/pending_predict/` | **待预测列表**：对单条或批量执行 **预测**（调用 `PredictService`，融合配置与模型以当前系统配置为准）。 |
| `risk/predictions/` | **预测结果列表**：按模型版本、风险等级、是否触发预警、时间范围筛选；支持导出 **CSV/JSON**（会写入 **导出审计日志**）。 |
| `risk/predictions/<id>/` | **预测结果详情**：查看分数、明细 JSON、关联预警等。 |
| `risk/alerts/` | **预警中心**：按状态、等级、时间筛选；支持导出 **CSV/JSON**（同样记审计日志）。 |
| `risk/alerts/<id>/` | **预警详情**：查看预警原因、原文、历史 **复核记录**。 |
| `risk/review/<alert_id>/`（POST） | **提交复核**：填写复核结论与说明后，预警状态会更新，必要时将对应 **`RawText`** 流转为 **已复核**。 |

### 2.6 模型管理

| 路径 | 功能 |
|------|------|
| `model/list/` | **模型列表**：查看已登记模型；**启用/停用** 模型（同时仅建议一个为启用）。 |
| `model/add/`、`model/edit/<id>/` | **新增/编辑模型元数据**：名称、类型（如 svm / textcnn）、版本、文件路径、向量化器路径、描述、JSON 配置与指标等。 |
| `model/active/` | **当前启用模型**：展示当前启用项及与其相关的运行时配置摘要。 |
| `model/config/` | **融合配置**：维护 **`SystemConfig`** 中的键值（如阈值、权重等），保存后对 **后续新产生的预测** 生效（具体键名以页面与 `PredictService.ensure_default_runtime_config` 为准）。 |
| `model/evaluation/` | **评估结果**：查看由训练脚本自动写入的历史评估记录（支持按模型筛选）。 |
| `model/algorithm_compare/` | **算法对比**：按算法类型汇总最新评估指标总表。 |
| `model/self_check/` | **模型自检**：对当前启用模型或全部模型执行健康检查；每次打开会生成一条 **自检历史** 记录。 |
| `model/self_check/history/` | **自检历史列表**：支持导出 CSV/JSON（记导出日志）。 |
| `model/self_check/history/<record_id>/` | **自检历史详情**：查看结构化/JSON 明细。 |

### 2.7 导出审计

| 路径 | 功能 |
|------|------|
| `audit/export_logs/` | **导出日志列表**：谁在何时以何种条件导出了预警、预测结果、自检历史等。 |
| `audit/export_logs/<log_id>/` | **单条导出日志详情**。 |

### 2.8 Django 自带后台（可选）

| 路径 | 功能 |
|------|------|
| `/admin/` | Django Admin，当前对业务 **`User`** 等做了简单注册，可用于直接改表（与 `admin_panel` 业务流无关）。需 **Django 超级用户** 登录。 |

## 三、用户端与管理员端数据关系（简表）

| 数据主线 | 说明 |
|----------|------|
| **Comment** | 爬虫 → CSV → `init` 写入；供前台列表、图表、在线预测演示、可选 `trigger_fenlei` 批量分类。 |
| **RawText → 标注 → 预测 → 预警 → 复核** | 后台导入/管理 **`RawText`**，在 **`admin_panel`** 内完成标注、预测、预警与复核闭环；导出操作记入 **`ExportLog`**。 |

两套数据可并存：前者偏「贴吧评论」展示与传统 ML 演示，后者偏「心理风险识别与预警辅助」完整业务链。

## 四、常见问题提示

1. **无法打开后台页或报与 `userinfo` 相关错误**：请先用 **前台 `/login/`** 登录 **`role=admin`** 的账号，再点「后台入口」。  
2. **待预测为空**：确认 **`RawText`** 是否已完成标注并进入待预测状态；模型是否已启用且文件路径有效。  
3. **导入无数据**：检查 CSV 是否含 **`content`** 列、编码是否为 UTF-8（建议带 BOM 的 `utf-8-sig`）。

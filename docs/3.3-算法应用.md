## 3.3.1 在线文本抑郁倾向预测的实现

### 1．在线文本预测

**功能描述：** 用户进入系统后访问在线预测功能（路由 `/predict/`），进入预测页面，在表单中输入待检测的中文文本，点击「确定」按钮，系统调用已训练的分类模型完成**抑郁倾向类别**（如：没有抑郁倾向、一般抑郁倾向、轻度抑郁倾向、严重抑郁倾向等，具体以 `model/num2name.json` 映射为准）的预测。在线预测页面，如图 3.3 所示。

**图 3.3 在线文本抑郁倾向预测页面**  
（此处插入 `predict.html` 实际运行截图：含「检测文本」输入框与「确定」按钮。）

**实现描述：** 在线预测由 Django 类视图 **`predict`**（定义于 `app/views.py`）承担。用户通过 **GET** 请求进入页面时，系统校验登录 Cookie 后使用 **`render()`** 渲染模板 **`templates/predict.html`**，展示检测文本输入区域。用户点击「确定」后，前端通过 **jQuery Ajax** 以 **POST** 方式向同一地址提交字段 **`content`**（检测文本），避免整页刷新。

服务端在 **`predict.post()`** 中完成算法链路：首先使用 **joblib** 加载磁盘上的 **`model/model_svm.pkl`**（支持向量机分类器）与 **`model/tfidfVectorizer.pkl`**（TF-IDF 向量化器），并读取 **`model/num2name.json`** 将模型输出的类别编号映射为中文标签。对输入文本依次进行**非中英文字符过滤**（正则）、**jieba 分词**、按 **`stopwords.txt`** 去除停用词，得到分词后的文本序列；再经 **TF-IDF 向量化** 与 **SVM 预测** 得到类别编号，通过 **`num2name`** 转为可读标签。最终将结果封装为 **`JsonResponse`** 返回前端（如 `status`、`msg` 字段），前端在 **`success` 回调**中以提示框等形式展示预测结论。

> **扩展说明（TextCNN）：** 若需采用卷积神经网络 TextCNN 进行推理，可将训练好的模型文件（如 **`model_textcnn.pkl`** 或框架对应的序列化格式）置于 **`model/`** 目录，并在上述预测分支中替换或并行加载模型与文本张量化流程；当前仓库主业务路径与上述描述一致，以 **SVM + TF-IDF** 实现。

**图 3.4 在线预测结果提示示意**  
（此处插入点击「确定」后弹出提示或页面内结果区域的截图。）

## 3.3.2 评论数据批量分类与多维度可视化的实现

本节对应系统中「对**已入库贴吧评论**进行规模化标注」以及「基于数据库统计结果进行图表展示」的算法与数据应用，在流程上与「先拉取用户行为再融合多路推荐」类似：本系统先汇聚**评论内容**，再经**统一文本管线**批量打标，最后在多个视图中做**统计与可视化**。

### 1．评论批量情感 / 抑郁倾向分类

**功能描述：** 对于已保存到数据库、**尚未写入情感标签**（`emotion` 字段为空）的评论记录，系统可执行批量分类，将每条评论映射到与在线预测一致的类别体系，并**回写数据库**，供后续首页列表、管理后台与统计图表使用。

**图 3.5 批量分类与数据准备流程示意**  
（可选：插入管理端评论列表或数据库中 `emotion` 字段更新前后对比截图。）

**实现描述：** 批量分类逻辑由 **`fenlei()`**（`app/views.py`）实现：查询 **`Comment.objects.filter(emotion='')`** 取出待标注评论，对 **`content`** 依次调用 **`clean_str()`**、**`sent2word()`**（内部使用 **jieba** 与 **`stopwords.txt`**）完成与训练阶段一致的文本规范化；再使用与在线预测相同的 **`model_svm.pkl`**、**`tfidfVectorizer.pkl`** 及 **`num2name.json`** 进行向量化和预测，通过 ORM **`update`** 将标签写入对应评论的 **`emotion`** 字段。  
此外，用户在**爬虫页**提交帖子 ID 后，**`scrawl`** 视图在 **`scrawl_tieba(tid)`** 抓取评论写入 **`data/{tid}.csv`** 后调用 **`init(tid)`** 将数据去重入库；新入库评论若尚未分类，可再通过上述批量流程统一标注，形成「**采集 → 入库 → 分类**」的数据链路。

---

### 2．多维度统计分析可视化

**功能描述：** 用户或管理员在登录后访问**词云、折线、环形、地图、柱状、散点**等可视化菜单（对应路由 **`plot1`～`plot7`**），系统从 **`Comment`** 模型中聚合已存储的评论文本、时间、情感标签、地域、用户名、等级、内容长度与回复数等字段，生成前端 **ECharts** 所需的数据结构并在模板中渲染，用于观察**抑郁倾向分布、时间趋势、地域分布**等。

**图 3.6 情感分类环形图等多维可视化页面示意**  
（此处插入 `plot3` 等页面实际截图。）

**实现描述：** **`plot1`** 对全量评论分词统计词频，供词云展示；**`plot2`** 按日期聚合评论数量；**`plot3`** 按 **`emotion`** 分组计数，供环形/饼图使用；**`plot4`** 按 **`loc`** 统计地域分布；**`plot5`**、**`plot6`** 分别按用户名、等级聚合；**`plot7`** 以评论长度与回复数构造散点数据。各视图在读取 Cookie 用户信息后，使用 **`render(request, 'plotN.html', locals())`** 将聚合结果传入模板，由静态资源中的 **ECharts** 完成绘图。管理员后台 **`admin_panel/statistics/`** 亦可展示统计类图表，与业务侧 **`plot*`** 形成**前台分析 + 后台监管**的补充。

| 对比项 | 车辆系统（示例） | 本抑郁分析系统 |

# 贴吧抑郁分析系统 — 项目说明

本文档整合了原「项目说明文档」、运行说明、管理员说明、登录注册更新说明及技术梳理要点，作为唯一主文档维护。

---

## 1. 项目概述

**题目**：面向社交媒体数据的抑郁检测与分析  

**技术栈**：Python 3.7、Django 2.2、Bootstrap、requests + XPath 爬虫、传统机器学习（SVM 等）与 TextCNN（模型文件存在，业务默认使用 SVM + TF-IDF）。

**数据库**：默认 SQLite（`db/db.sqlite3`，与 MySQL 可切换，见 `DjangoWeb/config.py`）。

**主要能力**：贴吧评论爬虫、数据入库、情感/抑郁倾向分类、多类图表可视化、在线文本预测、独立管理员前台。

---

## 2. 环境与运行

### 2.1 依赖安装

```bash
pip install -r requirements.txt
```

说明：`requirements-all.txt` 已移除；若需完整环境快照请自行 `pip freeze > requirements-lock.txt`，勿将无关环境整包混入仓库。

### 2.2 数据库

```bash
python manage.py makemigrations
python manage.py migrate
```

### 2.3 启动方式（任选）

- 推荐：`python manage.py runserver`，终端会显示本地访问地址。  
- 或使用项目根目录下的 `运行.py`（若你的 IDE 配置为从此启动）。

### 2.4 默认测试账号

- 普通用户：`user` / `123456`（若库中不存在，`init_user` 会尝试创建，以实际代码为准）。  
- Django 超级用户：需执行 `python manage.py createsuperuser` 自行创建。

### 2.5 使用注意

- 停止服务时：先在 IDE/终端停止运行，再关闭 PyCharm 等，避免进程残留。  
- 业务表在库中一般以 `app_` 为前缀，与 `app/models.py` 对应。

## 3. 目录结构（核心）

| 路径 | 说明 |
|------|------|
| `app/` | 业务代码：`views.py`（主逻辑）、`userViews.py`（登录注册）、`adminViews.py`（管理端）、`scrawldata.py`（爬虫） |
| `DjangoWeb/` | `settings.py`、`urls.py`、`config.py` |
| `templates/` | 页面模板；`templates/admin/` 为管理端 |
| `static/` | CSS/JS 等静态资源 |
| `media/` | 用户上传（如头像） |
| `model/` | 训练好的模型与 `tfidfVectorizer.pkl`、`num2name.json` 等 |
| `data/` | **唯一**爬虫与导入用 CSV 目录（已与原 `app/data` 合并，请勿再建重复目录） |
| `db/` | SQLite 数据文件 |
| `utils/` | 通用工具、MySQL 连接等 |
| `stopwords.txt` | 分词停用词 |

**算法与配置专档**：多模型融合、规则打分、训练评估、SystemConfig 键含义及**文档维护约定**见 **`docs/算法逻辑与配置说明.md`**。新增或修改预测/训练相关设定时请同步更新该文档（见其中 §0）。

## 4. 功能与路由（用户侧）

- 登录 `/login/`、注册 `/register/`  
- 首页帖子列表、爬虫页、可视化（词云、折线、情感环形图、地图、柱状图、散点等）  
- 在线预测页面（输入文本 → SVM + TF-IDF 预测）  

爬虫结果写入 **`项目根目录/data/{帖子tid}.csv`**，`init` 从该路径读入数据库。

---

## 5. 管理员系统

**入口**：`http://127.0.0.1:8000/admin_panel/login/`

| 路径 | 功能 |
|------|------|
| `/admin_panel/login/` | 管理员登录 |
| `/admin_panel/logout/` | 退出 |
| `/admin_panel/dashboard/` | 数据概览 |
| `/admin_panel/users/` | 用户管理（搜索、编辑、删除） |
| `/admin_panel/comments/` | 评论管理（搜索、按情感筛选、改标签、删除） |
| `/admin_panel/statistics/` | 统计图表（情感、地域、等级、趋势等） |

**创建管理员**：`python manage.py createsuperuser`，或使用符合后台校验规则的账号（以 `adminViews.py` 与数据库为准）。

**说明**：管理端为 Cookie 鉴权（有效期等以代码为准）；删除等操作需谨慎。

# 基于 TextCNN 与机器学习的语言文本抑郁分析系统 — 开发环境与项目文件结构说明

本项目路径下除各功能目录外，还在根目录集中存放工具脚本、运行入口、停用词及说明文档等，这些文件与目录共同支撑系统的开发、部署与稳定运行。系统以 Django 为 Web 框架，结合传统机器学习（如 SVM + TF-IDF）进行在线分类；**TextCNN 等深度学习模型文件可存放于 `model/` 目录供扩展接入**，与毕业论文/设计说明中的「深度学习分支」相对应。当前结构已完成一次轻量清理：移除 IDE 临时配置与未被业务引用的冗余工具文件，保持目录更聚焦于可运行代码与数据资源。

---

## 3.1 开发环境

| 类别 | 说明 |
|------|------|
| 操作系统 | Windows / Linux / macOS 均可，建议与团队部署环境一致 |
| Python | 建议 3.7（与项目依赖兼容；升级需自行验证 sklearn、Django 等版本） |
| Web 框架 | Django 2.2.x |
| 数据库 | 默认 SQLite（`db/db.sqlite3`）；可通过 `DjangoWeb/config.py` 切换 MySQL |
| 机器学习 | scikit-learn、jieba、joblib、numpy、pandas 等（见 `requirements.txt`） |
| 深度学习（扩展） | 项目依赖中含 TensorFlow/Keras 等；TextCNN 模型以 `.pkl` 等形式置于 `model/`，业务代码可按需加载 |
| 前端与可视化 | HTML 模板 + Bootstrap / Layui + ECharts（静态资源位于 `static/`） |
| 运行方式 | `python manage.py runserver` 或执行根目录 `运行.py`（内部调用上述命令） |

安装依赖示例：

```bash
pip install -r requirements.txt
python manage.py migrate
```

## 表 3.2 项目文件目录表

项目全部文件资源统一归档在 **DjangoWeb** 根目录下，作为系统顶层目录；各子目录与功能对应关系如下。

| 目录名 | 作用 |
|--------|------|
| `DjangoWeb/` | **项目配置包**：全局设置、总路由、WSGI、数据库配置等 |
| `app/` | **核心应用目录**：数据模型、用户视图、业务视图、管理员视图、爬虫逻辑等 |
| `static/` | 静态文件目录（CSS、JS、图片、第三方库如 Layui、ECharts 等） |
| `templates/` | HTML 模板目录（用户端、管理端、图表页、预测页等） |
| `media/` | 用户上传文件目录（如头像等，`MEDIA_ROOT` 指向此处） |
| `model/` | **模型与映射文件目录**：训练好的分类模型（如 `model_svm.pkl`）、`tfidfVectorizer.pkl`、**TextCNN 相关模型文件（如 `model_textcnn.pkl`，按实际文件名存放）**、`num2name.json` / `name2num.json` 等 |
| `data/` | 爬虫输出的 CSV 数据目录，供 `init` 导入数据库 |
| `db/` | SQLite 数据库文件存放目录（默认 `db.sqlite3`） |
| `utils/` | MySQL 连接与建库辅助模块（启用 MySQL 时使用） |
| `docs/` | 项目说明与设计文档目录 |

## 表 3.3 `DjangoWeb` 配置包内文件

核心配置集中在 **`DjangoWeb`** 包内（对应常见 Django 项目的「项目名」包）。全局参数在 `settings.py` 中集中配置（含应用注册、中间件、模板、静态/媒体路径等）；**数据库连接方式**在 `config.py` 中定义并由 `settings.py` 引用；**所有网页路径的分配规则**在 `urls.py` 中声明，负责将 HTTP 请求分发到 `app` 中各视图；`wsgi.py` 配置 Web 服务器网关接口，用于生产部署。

| 文件名 | 作用 |
|--------|------|
| `settings.py` | 项目全局配置：应用列表、中间件、数据库、国际化、静态文件与媒体路径等 |
| `urls.py` | 项目总 URL 路由：用户端、爬虫、可视化、预测、管理端面板等路径汇总 |
| `wsgi.py` | WSGI 入口，供 Gunicorn/uWSGI 等服务器挂载部署 |
| `config.py` | 数据库等环境相关配置（SQLite / MySQL 切换、MySQL 自动建库逻辑等） |
| `__init__.py` | Python 包标识文件 |

## 表 3.4 `app` 应用内功能划分说明

本项目采用 **单应用 `app`** 承载全部业务，在代码层面划分为：**用户与会话**、**文本采集与分析（含机器学习/TextCNN 扩展）**、**独立管理后台** 三类逻辑，分别由不同视图模块实现。

| 逻辑模块 | 作用 | 主要文件 |
|----------|------|----------|
| 用户与会话 | 注册、登录、注销、个人资料、修改密码 | `userViews.py`、`models.py`（User） |
| 文本与模型 | 贴吧爬虫、数据入库、情感/抑郁倾向分类、多图可视化、在线预测 | `views.py`、`scrawldata.py`、`models.py`（Comment） |
| 管理后台 | 管理员登录入口与后台会话校验（业务管理页由 `user_mgmt_views.py`、`riskViews.py`、`modelViews.py` 等承载） | `adminViews.py` |

各模块均通过 `DjangoWeb/urls.py` 挂载路由；数据模型在 `models.py` 中定义，Django 默认 `admin.py` 可用于自带后台（本项目另有 `admin_panel` 前端）。

## 表 3.6 文本分析、爬虫与可视化相关文件

本模块对应**语言文本抑郁分析**的主流程：采集贴吧评论 → 入库 → **基于机器学习（当前实现为 SVM + TF-IDF）批量分类** → 多维度可视化 → **在线文本预测**（可扩展为 TextCNN 推理）。

| 文件名 | 作用 |
|--------|------|
| `scrawldata.py` | 贴吧评论爬虫：会话与 Cookie、分页抓取、写入 `data/{帖子tid}.csv` |
| `views.py` | 首页列表、爬虫页、`init` 数据导入、`fenlei` 分类逻辑、plot1–plot7 图表数据、`predict` 在线预测等 |
| `models.py`（Comment 部分） | 评论实体：时间、楼层、昵称、链接、头像、等级、内容、回复数、性别、地域、`emotion` 等 |
| `management/commands/*.py` | Django 管理命令扩展（如预测回填、数据清理、评估清理等） |
| `data/*.csv` | 爬虫原始数据，供导入数据库 |
| `model/model_svm.pkl` | 当前业务默认使用的 SVM 分类模型（文件名以仓库实际为准） |
| `model/tfidfVectorizer.pkl` | TF-IDF 向量化器，与 SVM 管线配合 |
| `model/model_textcnn.pkl` | TextCNN 模型文件（**扩展用**；若仓库中已训练保存则置于此，需在 `views.py` 中切换加载逻辑方可生效） |
| `model/num2name.json`、`name2num.json` | 类别编号与情感/抑郁倾向标签名称映射 |
